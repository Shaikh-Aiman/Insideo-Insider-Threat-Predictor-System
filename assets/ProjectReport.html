INSIDEO - Insider Threat Predictor

Definition

Insider Threat Predictor is a data-driven security system designed to identify and analyze potentially malicious or risky activities performed by internal users. It leverages log aggregation, behavioral feature engineering, anomaly detection, and risk scoring techniques to detect deviations from normal behavior. The system supports predictive analytics and visualization to proactively mitigate insider threats in organizational environments.

Core Features

  1. User Activity Aggregation
    Collects and aggregates user activity logs such as login attempts, file access, and data transfers on a per-user, per-day basis. This aggregation helps establish individual behavioral baselines for accurate insider threat analysis.

  2. Behavioral Feature Engineering
    Extracts meaningful statistical and temporal features including event frequency, resource diversity, failed login counts, and deviations from historical behavior to characterize user activity patterns.

  3. Anomaly Detection using Machine Learning
    Applies unsupervised machine learning models such as Isolation Forest to identify abnormal user behavior that deviates significantly from established norms without requiring labeled attack data.

  4. Risk Scoring Engine
    Dynamically computes user risk scores using weighted security-relevant actions and predefined threat rules, enabling continuous assessment of insider threat levels.

  5. Incident Detection and Logging
    Automatically flags high-risk users, generates security incidents when thresholds are exceeded, and maintains detailed audit logs to support investigation and forensic analysis.

  6. Interactive Visualization Dashboard
    Provides an intuitive SOC-style graphical interface with charts, tables, and pop-up controls for monitoring risks, anomalies, alerts, and overall system status in real time.

  7. Scalable and Modular Architecture
    Designed with modular components for data ingestion, analytics, and visualization, allowing easy extension with advanced models, new data sources, or additional security modules.

How To Operate

  1. Launch the Application
    Run main.py. The Insider Threat Predictor dashboard opens with the home screen and sidebar.

  2. Load or Generate Data
    Load a real-world CSV event log or generate a synthetic sample dataset to simulate insider activity for testing and validation.

  3. Recompute Aggregates
    Aggregate user activities on a per-user, per-day basis to build behavioral baselines.

  4. Compute Enriched Features
    Temporal, behavioral, statistical, and graph-based features are calculated to enhance detection accuracy.

  5. Run Risk Scoring and Anomaly Detection
    Rule-based risk scoring is applied, and the Isolation Forest model detects anomalous user behavior. Detected events are mapped to the MITRE ATT&CK framework and visualized using a heat map.

  6. Train LSTM Model (User-Defined Lookback)
    Select the number of lookback days and train the LSTM model to learn historical behavior patterns.

  7. Insider Risk Prediction using LSTM
    The trained LSTM model predicts insider risk probabilities, classifies users, and generates alerts for high-risk behavior.

  8. Results Visualization and Export
    Risk scores, anomaly reports, and LSTM alerts are displayed through tables and charts. Reports can be exported as CSV files for further analysis or documentation.

  9. Clear Data and Reset System
    Clear all loaded data, model outputs, and internal states to reset the system for a fresh analysis cycle.

Technologies & Libraries Used

  1. threading
    Used to run time-consuming tasks such as event processing, Isolation Forest execution, and LSTM training in parallel, ensuring the Tkinter GUI remains responsive without freezing during heavy computations.

  2. traceback
    Captures and displays detailed error stack traces during runtime exceptions, allowing graceful debugging without crashing the application.

  3. datetime
    Handles timestamp parsing, event ordering, daily aggregation, and incident logging to enable time-based behavioral analysis.

  4. tkinter
    Builds the complete desktop-based SOC-style graphical interface including dashboards, popups, buttons, and layout components.

  5. tkinter.ttk
    Provides advanced widgets such as Treeview tables, scrollbars, and structured layouts for improved visualization and usability.

  6. tkinter.filedialog
    Enables loading CSV datasets and exporting anomaly or risk reports without hardcoded file paths.

  7. tkinter.messagebox
    Displays alerts, warnings, confirmations, and system status messages during application runtime.

  8. pandas
    Performs data ingestion, cleaning, aggregation, feature engineering, and preparation of structured inputs for machine learning models.

  9. numpy
    Supports numerical operations, array manipulation, and efficient handling of LSTM input sequences.

  10. random
    Generates realistic synthetic insider threat datasets for testing, experimentation, and validation.

  11. scikit-learn (Isolation Forest)
    Performs unsupervised anomaly detection on aggregated user behavior features to identify deviations from normal activity.

  12. scikit-learn (StandardScaler)
    Normalizes numerical features to prevent scale bias during anomaly detection and model evaluation.

  13. matplotlib
    Generates visualizations such as risk score plots and MITRE ATT&CK heat maps.

  14. matplotlib.backends.backend_tkagg
    Embeds Matplotlib charts directly into the Tkinter GUI for interactive visualization.

  15. Pillow (PIL.Image, ImageTk)
    Loads, resizes, and displays images such as logos and home screen visuals within the GUI.

  16. TensorFlow / Keras (Sequential)
    Builds and manages the deep learning architecture used for insider threat prediction.

  17. Keras LSTM Layer
    Learns temporal behavior patterns from historical user activity sequences.

  18. Keras Dense Layer
    Produces final probability scores indicating insider threat likelihood.

  19. Keras Dropout
    Prevents overfitting and improves model generalization.

  20. Keras Adam Optimizer
    Optimizes LSTM training for faster and stable convergence.

  21. os
    Handles file existence checks, directory paths, and report storage operations.

  22. networkx
    Computes graph-based metrics such as user-resource relationships and centrality scores.
